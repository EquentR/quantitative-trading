# A 股短线量化建议系统项目规约

## 1. 项目定位

本项目是一个面向个人使用的 A 股短线量化决策辅助系统。系统目标是在不自动下单、不控制真实交易客户端的前提下，读取手动持仓台账、自选置顶股票和行情数据，生成可解释的买入、卖出、加仓、减仓、观望建议，并通过本地输出和后续消息渠道提醒用户。

系统只做个人研究和决策辅助，不承诺正收益。所有真实交易必须由用户在交易软件中人工确认。

## 2. 首版目标

首版以“能稳定生成可追溯建议”为核心，不追求复杂模型和自动化交易。

- 维护并读取手动持仓台账，将其作为真实持仓的唯一权威源。
- 读取东方财富自选置顶股票，作为候选买入观察池。
- 使用 AkShare 等公开数据源获取行情、历史 K 线、技术指标、资金流和分时强弱数据。
- 收盘后生成次日交易计划。
- 盘中仅对持仓和计划内股票做触发判断。
- 输出本地控制台消息和结构化日志。
- 后续支持 Telegram、飞书、钉钉等 API 友好的推送渠道。
- 邮件仅作为兜底告警渠道。
- 东方财富模拟组合可在后续作为自动模拟操作决策层，用于演练策略建议，不作为真实持仓来源。

## 3. 非目标

以下能力不属于首版范围：

- 自动下单、模拟点击交易客户端、绕过人工确认。
- 直接读取真实交易账户的敏感信息。
- 依赖东方财富模拟组合推断真实持仓成本、数量或可用数量。
- 使用无法解释的黑盒模型作为主要决策依据。
- 全市场高频扫描。
- Web UI、移动 App、复杂可视化看板。
- 承诺收益率、胜率或资金安全。

## 4. 运行形态

项目必须同时支持后台常驻服务和 CLI 操作入口，不能只支持其中一种。

后台服务负责：

- 按交易日时间表运行。
- 收盘后生成次日计划。
- 盘中定时检查触发条件。
- 写入审计日志。
- 发送本地和远程推送。

CLI 负责：

- 检查配置和数据源状态。
- 手动新增、更新、删除、导入和查看持仓台账。
- 手动同步东方财富自选置顶列表。
- 手动拉取行情数据。
- 生成某日交易计划。
- 查看当前建议。
- 回放某日数据用于复盘。
- 手动触发一次性扫描。

后台服务和 CLI 必须共享同一套核心逻辑。不得为后台和 CLI 分别实现两套策略、风控或数据解释逻辑。

## 5. 股票池

首版股票池只包含：

1. 手动持仓台账中的股票。
2. 东方财富自选置顶股票。

当前持仓来自手动持仓台账。持仓股票优先级高于自选置顶股票。持仓股票用于卖出、止损、止盈、加仓、减仓和观望建议；自选置顶股票主要用于候选买入观察。

## 6. 决策节奏

系统采用“收盘计划 + 盘中触发”的节奏。

收盘后：

- 读取手动持仓台账和自选置顶列表。
- 拉取历史行情、技术指标和资金流数据。
- 生成次日观察列表、关键价位、候选动作和风险条件。

盘中：

- 只对持仓股票和计划内股票做准实时检查。
- 根据行情、分时强弱、资金流和关键价位判断是否触发建议。
- 所有建议必须先通过风控规则，再进入推送。

## 7. 核心模块

### Data Adapters

封装外部数据源，包括 AkShare、东方财富自选置顶接口，以及后续备用数据源。外部接口变化应尽量只影响 adapter。

### Manual Position Ledger

维护当前真实持仓的手动台账。手动持仓台账是系统计算仓位、成本、止损、止盈和持仓风险的唯一权威来源。

持仓台账至少应包含股票代码、名称、持仓数量、可用数量、成本价、建仓日期、最近更新时间和备注。后台服务只读取持仓台账，CLI 负责手动维护持仓台账。

系统不得从东方财富模拟组合推断真实持仓成本、数量或可用数量。建议中必须保留持仓台账更新时间和行情数据时间。

### Universe Builder

生成首版股票池，并保留来源标记。股票来源至少包括 `holding` 和 `watch_pinned`。

### Market Data Cache

缓存持仓台账快照、行情快照、历史 K 线、资金流、分时强弱、计划和建议。缓存用于减少重复请求，并保证复盘时可以追溯当时使用的数据。

### Strategy Engine

生成候选信号。首版采用规则策略，以技术面为主，资金流和分时强弱作为确认或过滤条件。策略输出必须可解释。

### Risk Engine

在最终建议前执行硬性风控约束。风险约束优先于策略信号。

### Recommendation Engine

合成最终建议，统一字段格式，并确保每条建议都包含理由、风险、失效条件、仓位约束和数据时间戳。

### Notifier

负责输出建议。首版必须支持本地控制台和结构化日志；第二阶段支持 Telegram、飞书、钉钉；邮件作为兜底告警。

### Simulation Operation Layer

预留东方财富模拟组合接口作为自动模拟操作决策层。该层可在后续接收系统建议，并在模拟组合中执行模拟买入、模拟卖出或组合调整，用于演练和复盘策略表现。

模拟操作决策层不得修改手动持仓台账，不得代表真实交易，不得影响真实持仓建议的成本和数量计算。

### Audit Log

记录输入数据、信号、风控结果、最终建议、推送结果和人工反馈。审计日志用于复盘、排错和后续回测。

## 8. 技术栈约束

首版按 Python 项目设计。

- Python 3.11 或更高版本。
- 包管理优先考虑 uv。
- 数据处理优先使用 pandas。
- 数据结构和配置校验优先使用 pydantic。
- 后台调度可使用 APScheduler 或同类成熟调度器。
- 代码应为后续测试、回放和替换数据源保留边界。

## 9. 成功标准

首版完成后，应能做到：

- 通过 CLI 检查配置、维护持仓台账、同步数据、生成计划、查看建议。
- 通过后台服务在交易日自动执行计划生成和盘中触发检查。
- 对每条建议给出可解释理由和风险条件。
- 对建议全过程留下可追溯日志。
- 在不接入自动交易的情况下支持用户手动决策。
