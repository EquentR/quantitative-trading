# ETF 目录名称兼容修复设计

## 背景与根因

证券搜索预览依赖本地标准化沪深 A 股和 ETF 目录。ETF 必须通过公开 spot 目录、交易所规模目录和交易所分类目录交叉验证，才能作为已验证 ETF 加入观察池并路由到 ETF 行情 provider。

`512480` 在公开目录中的代码一致，但简称不同：

- AkShare 东方财富 ETF spot：`半导体ETF国联安`。
- 上交所 ETF 规模目录：`半导体`。

当前名称兼容规则只接受“共同名称已经包含 ETF，再追加管理人简称”的形式。它错误拒绝了“交易所主题简称 + ETF + 管理人简称”的常见形式，将 `512480` 保存为 `instrument_type=unknown` 和 `settlement_cycle=unknown`。因此搜索预览不可选择，持仓只能人工复核，盘中工作流也不会把该代码交给 ETF 报价 provider，最终保存 `provider did not return requested quote`。

## 目标

- 接受结构明确的 ETF 长短简称差异，不为单只证券维护硬编码别名。
- 搜索预览、观察池和行情页使用 spot 目录的较完整名称，便于区分同主题、不同管理人的 ETF。
- 保持代码、交易所、证券类型和交易制度的权威验证门禁。
- 目录刷新后让已有持仓自动恢复 ETF 行情路由，不从持仓名称或代码前缀猜测类型。

## 非目标

- 不把任意同代码名称差异都视为兼容。
- 不按名称关键词推断 ETF、交易所或 T+0/T+1。
- 不修改手动持仓台账中的数量、成本或可用数量。
- 不增加自动交易或真实下单能力。

## 名称兼容规则

名称比较继续忽略空白并统一 ASCII 大小写。以下情况视为兼容：

1. 两个名称完全相同。
2. 较短名称已经包含 `ETF`，较长名称以较短名称开头；较长部分视为管理人等展示后缀。
3. 较长名称以较短名称开头，且剩余部分以 `ETF` 开头；这覆盖 `半导体` 与 `半导体ETF国联安`。

其它差异仍视为冲突。例如 `半导体` 与 `半导体50ETF` 不兼容，因为新增部分不是以 `ETF` 开头。名称兼容只在相同六位代码已同时出现在所需目录后执行，不能替代代码交叉验证。

验证通过时，内部 `InstrumentMetadata.name` 使用 spot 目录名称。这样 `512480` 保存为 `半导体ETF国联安`，而不是较短且容易与其它产品混淆的 `半导体`。spot 名称缺失、重复冲突或与交易所简称不满足上述规则时，继续保存 `unknown` 和明确 warning。

## 数据流与恢复

搜索接口仍按现有流程执行：

1. 确保本地证券目录当日可用。
2. adapter 对多个公开目录做代码、名称和分类交叉验证。
3. repository 原子替换标准化目录。
4. 搜索服务从本地目录创建十分钟预览。
5. 用户人工确认后才增量加入观察池。

修复部署后显式失效一次当天目录缓存并执行刷新。刷新会把 `512480` 从 `unknown` 更新为已验证 ETF，并保存 spot 完整名称及交易所分类解析得到的结算制度。后续 `DecisionWorkflow` 通过现有元数据 loader 自动把持仓代码分配给 ETF 报价、日 K 和分钟线 provider，不增加特殊路由。

## 错误处理

- 真正的名称冲突继续返回 `ETF <symbol> has conflicting directory names`，候选不可选择。
- spot 目录缺少代码、名称缺失或同代码多名称冲突时，继续保持 `unknown`。
- `FUND_LIST` 缺失、类别冲突或规则未覆盖时，不因名称兼容而伪造交易制度；按现有规则保持观察限制或 `unknown`。
- 目录刷新失败时继续使用现有 stale 目录语义，不破坏观察池和持仓台账。

## 测试与验证

- adapter 回归测试使用 `512480` 的真实名称组合，先证明旧规则产生 `unknown`，再验证修复后为 ETF、使用完整 spot 名称并保留正确结算制度。
- 保留真实冲突用例，验证 `半导体` 与 `半导体50ETF` 仍为 `unknown`。
- 搜索/候选测试验证完整名称可搜索、候选可选择。
- 工作流测试验证刷新后的元数据把 `512480` 路由到 ETF quote provider，而不是产生 `provider did not return requested quote`。
- 运行相关后端测试、完整后端套件、前端测试和生产构建。
- 刷新本地真实目录后检查 `instruments`、搜索 API 和下一轮盘中 capture result，确认证券类型、结算制度、完整名称及 ETF 报价状态。
