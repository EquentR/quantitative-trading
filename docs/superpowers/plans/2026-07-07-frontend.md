# 前端控制台实现计划

> **给 agent worker 的要求：** 必须使用子技能：推荐使用 `superpowers:subagent-driven-development`，或使用 `superpowers:executing-plans` 按任务执行本计划。步骤使用复选框（`- [ ]`）跟踪。

**目标：** 基于已确认的前端设计，实现一个 Vue 3 本地量化控制台，对接后台 `/api/v1` HTTP API，覆盖认证/setup、今日仪表盘、准备、监控、复盘、设置，以及手动持仓台账和手动资金账户维护能力。

**架构：** 在现有 Python 仓库内新增独立 `frontend/` Vite 子项目，不改动后端核心业务逻辑。前端按 API client、query hooks、session store、应用布局、业务页面分层；测试使用 MSW mock `/api/v1`，不依赖真实后端、AkShare 或网络行情。

**技术栈：** Vue 3、Vite、TypeScript、Tailwind CSS、shadcn-vue/Radix Vue 风格组件、TanStack Query for Vue、Pinia、Vue Router、zod、vee-validate、Vitest、Testing Library、MSW、Playwright。

---

## 范围与安全约束

本计划实现 `docs/superpowers/specs/2026-07-07-frontend-design.md`。

执行时必须遵守：

- 不实现自动真实下单。
- 不模拟点击或控制真实交易客户端。
- 不读取、保存或提交真实券商账号、密码、cookie、token 或 API key。
- 持仓写操作只能表达为“维护手动持仓台账”。
- 资金写操作只能表达为“维护手动资金账户”。
- 用户可见成功文案不得使用“买入成功”“卖出成功”“成交”“委托”“提现成功”等真实交易语义。
- 复盘页第一阶段只做后续能力预留，不展示假建议、假反馈或假收益。
- 前端是本地单用户控制台，不实现多用户、租户或角色权限。

创建 `frontend/src/stores/session.ts`：

```ts
import { defineStore } from 'pinia'

const API_BASE_URL_KEY = 'qt_console_api_base_url'
const ACCESS_TOKEN_KEY = 'qt_console_access_token'

export const useSessionStore = defineStore('session', {
  state: () => ({
    apiBaseUrl: localStorage.getItem(API_BASE_URL_KEY) ?? '',
    token: localStorage.getItem(ACCESS_TOKEN_KEY)
  }),
  actions: {
    setApiBaseUrl(value: string) {
      const normalized = value.replace(/\/$/, '')
      this.apiBaseUrl = normalized
      localStorage.setItem(API_BASE_URL_KEY, normalized)
    },
    setToken(value: string) {
      this.token = value
      localStorage.setItem(ACCESS_TOKEN_KEY, value)
    },
    clearToken() {
      this.token = null
      localStorage.removeItem(ACCESS_TOKEN_KEY)
    }
  }
})
```
