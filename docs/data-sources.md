# 数据源规约

## 1. 数据源分层

首版数据源按以下优先级设计：

1. 手动持仓台账：作为真实持仓的唯一权威来源。
2. 手动资金账户：作为现金余额、净本金和可用买入资金的唯一权威来源。
3. 东方财富自选置顶接口：作为候选买入股票池来源。
4. AkShare：用于行情、历史 K 线、资金流、板块和其他公开市场数据。
5. 本地缓存：用于降低请求频率、支持追溯和复盘。
6. 东方财富模拟组合接口：后续作为自动模拟操作决策层，不作为真实持仓或真实资金来源。
7. 后续备用 adapter：用于替换不稳定或不可用的数据源。

## 2. 手动持仓台账

手动持仓台账是系统计算真实持仓、成本、数量、可用数量、仓位比例和持仓风险的唯一权威来源。

台账必须支持手动新增、更新、删除、导入和查看。CLI 是维护台账的主要入口，后台服务只读取台账，不自动修改真实持仓信息。

台账至少应记录：

- 股票代码。
- 股票名称。
- 持仓数量。
- 可用数量。
- 成本价。
- 建仓日期。
- 最近更新时间。
- 备注。

系统生成持仓相关建议时，必须记录使用的台账版本或更新时间。若持仓台账缺少成本、数量或可用数量等关键字段，相关建议应降级或暂停。

## 3. 手动资金账户

手动资金账户是系统计算现金余额、累计转入、累计转出、净本金、可用买入资金和账户仓位比例的权威来源。

资金账户必须支持初始化本金、模拟银证转入、模拟银证转出、现金校准和交易流水查询。CLI 是首版维护入口，未来 API 应复用同一套服务逻辑；后台服务只读取资金账户并写入账户快照，不得自动修改资金账户。

账户估值由手动持仓台账、手动资金账户和 AkShare 行情快照共同生成。行情缺失或异常时，系统必须显式标记快照状态，不能用缺失行情下的估算值替代完整账户估值。

## 4. 东方财富自选置顶

自选置顶股票用于首版候选买入观察。系统不扫描全部自选股，只使用置顶集合，以降低噪音和数据压力。

自选置顶股票应保留排序或优先级信息，便于策略在多候选股票中排序。

当前实现维护一份本地“自选置顶观察池”镜像，字段为 `symbol`、`name`、`rank`、`plan_enabled` 和 `note`，并额外记录 `source` 与 `updated_at`。`source` 取值包括：

- `manual`：用户通过 CLI 或 HTTP API 手动维护。
- `synced`：从外部自选置顶来源同步产生，且没有本地人工覆盖。
- `manual_synced`：同一股票既存在外部同步结果，也保留了本地人工设置。

`plan_enabled` 控制自选置顶股票是否进入收盘计划和盘中决策。持仓股票无论是否也在自选置顶中，都按持仓来源进入计划，`plan_enabled=true` 且 `plan_enabled_source=holding`。非持仓自选股票只有在 `plan_enabled=true` 时才进入 `watch_symbols`；同步新增的自选股票默认 `plan_enabled=false`，需要人工开启后才参与计划。

第三方同步不得覆盖本地维护的 `plan_enabled` 和 `note`。当同步结果中不再出现某个纯 `synced` 股票时，本地镜像会删除该记录；当 `manual_synced` 股票不再出现在同步结果中时，会降级为 `manual` 并保留本地设置。

## 5. 东方财富模拟组合

东方财富模拟组合不作为真实持仓或真实资金来源。系统不得依赖模拟组合推断真实持仓成本、数量、可用数量、现金余额或净本金。

用户后续可能提供东方财富模拟组合操作 Skill、接口和 API key。接入后，该能力应被视为自动模拟操作决策层，可用于：

- 将系统建议同步到模拟组合做演练。
- 根据建议执行模拟买入、模拟卖出或模拟调仓。
- 记录策略在模拟账户中的表现。
- 对比模拟操作结果和真实人工执行反馈。

模拟操作结果不得自动写回手动持仓台账。若需要调整真实持仓台账，必须由用户通过 CLI 手动确认。

## 6. 东方财富妙想 Skill 包

东方财富妙想 Skill 包可作为项目内供应商能力资料保存，用于后续封装东方财富金融数据、资讯搜索、智能选股、自选股管理、模拟组合管理和 AI 社区相关 adapter。

当前下载的 Skill 包位于 `vendor/miaoxiang/skills/`，仅表示项目保存了可参考或可封装的第三方能力文件，不表示已安装到 Codex、ClawHub 或其他 agent 运行时。

使用这些能力时必须遵守以下约束：

- 真实持仓、成本、数量和可用数量仍以手动持仓台账为唯一权威源。
- 现金余额、净本金和可用买入资金仍以手动资金账户为唯一权威源。
- 第三方接口调用必须由项目 adapter 封装，策略、风控和推荐模块不得直接依赖 Skill 文件或外部接口字段。
- `MX_APIKEY` 只能通过环境变量或未入库本地配置读取，不得写入 git。
- 东方财富模拟组合相关能力只能用于策略演练和复盘，不得替代真实人工执行反馈或自动修改手动持仓台账。

东方财富和妙想相关密钥必须按以下口径处理：

- 示例、测试、文档只能使用占位符，不得出现真实 API key、token、cookie 或账号。
- `MX_APIKEY` 只允许来自环境变量或未入库本地配置。
- `/api/v1/datasource/eastmoney/key` 只用于维护本地数据源凭据状态，API 响应只返回 `configured`、`missing` 或 `invalid` 等状态，不返回密钥明文。
- 前端输入框不得把 API key 写入 Pinia、localStorage 或结构化日志；提交失败时也不得清空或泄露用户输入。

## 7. AkShare

AkShare 是首版公开市场数据的首选来源。可用于：

- 日线、分钟线、历史 K 线。
- 实时或准实时行情。
- 成交额、成交量、涨跌幅。
- 资金流数据。
- 板块、指数、市场环境数据。

AkShare 数据可能存在延迟、字段变化、接口不可用或限频问题。adapter 必须显式处理异常，不得让错误数据静默进入策略。

AkShare 只提供行情报价和公开市场数据，不提供真实持仓、成本、数量、可用数量、现金余额或净本金。核心策略和账户服务不得直接依赖 AkShare 原始字段，必须通过项目内部 quote adapter 和统一模型访问。

当 AkShare 的批量实时行情接口不可用时，行情 adapter 可以在同一 adapter 边界内使用东方财富公开单票行情接口和腾讯公开 quote 接口作为兜底，只返回项目内部 `QuoteSnapshot` 字段，并保留数据源和失败告警。兜底接口不得绕过 adapter 进入账户服务、策略或风控逻辑。

## 8. 行情决策数据与可追溯快照

### 8.1 标的范围与共享采集

重数据和决策只覆盖手动持仓，以及 `plan_enabled=true` 的非持仓自选标的；同一股票同时来自两种来源时只采集一次并保留两种来源。盘中范围进一步收窄为当前持仓和当日活动计划内标的。provider 返回的额外股票必须忽略并告警，不能借外部响应扩大股票池。

报价按 provider 能力对本轮标的批量调用一次。日 K 和资金流只在首次回填及收盘校正窗口采集，分钟线只请求当前交易日。账户估值、策略、风控、Web 刷新不得绕过 `DecisionWorkflow` 重复调用第三方。`close/intraday/backfill/cleanup` 四类运行共享这一应用服务；旧建议 scan 入口不得形成第二条采集或写建议路径。每轮记录 provider 调用次数和耗时、请求/处理标的数、返回/写入/清理行数、逐数据集状态计数和失败数。

### 8.2 Adapter 与交易日历

市场模块通过内部 Protocol 提供报价、前复权日 K、逐股资金流和 1 分钟行情。AkShare 是四类数据的首选公开来源；东方财富公开单票行情和腾讯 quote 兜底只用于报价 adapter，不自动扩展为历史、资金流或分钟数据兜底。

adapter 必须完成第三方字段映射、数值和单位统一、六位股票代码校验、时区和交易日校验，只返回项目内部模型。日历使用 `exchange-calendars` 的 `XSHG`；下一交易日、交易分钟、午休、stale 计算和保留期边界都不得用自然日或简单周一至周五近似。

### 8.3 前复权日 K

每个决策启用标的首次补齐最近 250 个 `XSHG` 交易日，之后增量采集并刷新最近 5 个交易日的校正窗口。数据统一为 `adjustment=forward`，未经复权或后复权数据不得混入同一特征窗口。

日 K 事实按 `symbol + trade_date + adjustment + content_hash` 追加版本；provider 修正历史值时新增版本，不覆盖旧事实。每次收盘或回填创建不可变 `history_snapshot` 及 members，引用当轮实际使用的事实版本。日 K 和历史快照本期不自动删除，API 默认最多返回最近 250 个交易日。

### 8.4 完整资金流

资金流保存主力、超大单、大单、中单和小单的净流入额与净占比。金额统一为人民币元，占比统一为百分点数值；所有档位必须通过 adapter 映射，核心策略不能依赖 AkShare 原始列名。

每个启用标的首次补齐最近 60 个 `XSHG` 交易日，之后增量采集并创建不可变 `money_flow_snapshot` 及 members。事实按 `symbol + trade_date + content_hash` 追加版本；本期不自动删除，API 默认最多返回最近 60 个交易日。资金流只能作为确认或过滤条件，不能单独触发买卖动作。

### 8.5 分钟线和分时强弱

盘中保存当前交易日 1 分钟 OHLC、成交量和成交额，`minute` 必须位于 `09:30-11:30` 或 `13:00-15:00 Asia/Shanghai` 的有效交易分钟。唯一键为 `symbol + minute`，同一分钟幂等更新。成交量必须在 adapter 层统一为股，后端才可据此计算 VWAP 和分钟量比。

原始分钟线滚动保留最近 20 个 `XSHG` 交易日，并在交易日 `16:35` 独立清理。清理不得删除已经生成的分时强弱快照、规则版本、建议输入、数据摘要或审计记录。分时强弱必须保存组件值、方向、理由、覆盖率、最后分钟、实际阈值和规则版本；前端只展示这些后端事实。

### 8.6 市场输入快照与质量

每轮 `DecisionWorkflow` 生成 `run_id` 和不可变 `MarketInputSnapshot`，包含 `quote_snapshot_refs`、`history_snapshot_refs`、`money_flow_snapshot_refs`、`intraday_strength_snapshot_refs`、逐标的逐数据集质量、覆盖范围、`data_time`、`fetched_at` 和 warnings。计划、策略、风控、建议、通知和审计只能引用本轮已经固化的输入。

盘中聚合 `data_time` 取本轮有效报价和分时强弱时间的最早值，避免较新单条数据掩盖旧数据；`fetched_at` 是系统获取时间，不能替代市场时间。stale 默认阈值为 6 个有效交易分钟，由 `QT_MARKET_STALE_TRADING_MINUTES` 配置，午休和非交易时间不累计；实际生效阈值随快照保存并由 trace API 返回。稀疏响应和逐标的失败必须持久化为 `failed/degraded/stale` 结果，不得用空映射冒充成功。数据库引用、交易日或模型契约失败应终止整轮。

capture run 还必须保存工作流类型、交易日、周期、幂等键、开始/结束时间和成本指标，并按 `quote/daily_bar/money_flow/minute_bar/intraday_strength` 汇总 `complete/degraded/failed/stale` 的 `dataset_counts`。监控只展示这些已保存事实，不因查看页面再次触发 provider。

CLI、认证 HTTP API 和调度器共享同一 adapter、repository 和工作流，不输出第三方原始响应。所有快照只用于决策辅助和追溯，不代表真实成交，也不读取或修改真实券商账户。

## 9. 本地缓存

系统应缓存以下信息：

- 手动持仓台账快照。
- 手动资金账户快照。
- 账户估值快照。
- 自选置顶快照。
- 行情快照。
- 版本化前复权日 K、历史快照及成员引用。
- 完整资金流事实、快照及成员引用。
- 原始分钟线和不可变分时强弱快照。
- 工作流 capture run、逐标的数据集结果和市场输入快照。
- 收盘计划。
- 盘中触发结果。
- 最终建议。
- 推送结果和人工反馈。
- 模拟组合操作记录。

缓存必须保留数据时间戳和拉取时间。复盘时应能区分“市场数据发生时间”和“系统获取时间”。

## 10. HTTP API 写操作口径

HTTP API 对持仓和资金的写操作等价于用户维护手动持仓台账和手动资金账户。API 不读取真实券商账户，不从 AkShare、东方财富模拟组合或其他外部数据源推断真实持仓或真实资金。

账户快照接口和调度任务可以读取手动持仓台账、手动资金账户和行情数据，并写入本地账户快照。快照结果只用于本地估值、审计和后续建议上下文，不代表真实券商账户状态。

自选置顶 API 只维护本地观察池镜像，不代表真实持仓或真实交易。人工执行反馈 API 只写入复盘反馈和关联通知状态，不得修改手动持仓台账、手动资金账户、现金余额或净本金。

## 11. 密钥管理

API key、token、cookie、账号信息和任何敏感凭据不得提交到 git。

实现阶段应使用本地环境变量、未入库配置文件或系统密钥管理方式读取敏感信息。示例配置只能包含占位符。

SMTP 密码是本期唯一的秘密存储例外：经用户确认后允许明文写入本地 SQLite，但读取 API、CLI、Web 回填、日志、审计、通知、错误响应和邮件 outbox 永远不得返回或记录原值。数据库、备份和本地运行数据必须保持 git 忽略并限制为本机用户可读；导出或备份数据库会包含 SMTP 明文密码，界面和操作文档必须明确提示该风险。SMTP 未配置、禁用或失败不得阻止数据库/Web 通知、控制台和 JSONL 本地通道；永久失败邮件必须通过这些本地通道产生脱敏告警。

## 12. Adapter 约束

所有外部数据源都应通过 adapter 访问。策略、风控和推荐模块不得直接调用 AkShare 或东方财富接口。

adapter 输出应尽量转成项目内部统一模型，避免外部字段名泄漏到核心逻辑。

手动持仓台账和手动资金账户不是外部行情源，但也应通过清晰的 repository 或 storage 接口访问，避免 CLI、后台和策略各自读写不同格式。
