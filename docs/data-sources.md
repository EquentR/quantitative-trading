# 数据源规约

## 1. 数据源分层

首版数据源按以下优先级设计：

1. 东方财富模拟组合接口：作为持仓镜像来源。
2. 东方财富自选置顶接口：作为候选买入股票池来源。
3. AkShare：用于行情、历史 K 线、资金流、板块和其他公开市场数据。
4. 本地缓存：用于降低请求频率、支持追溯和复盘。
5. 后续备用 adapter：用于替换不稳定或不可用的数据源。

## 2. 东方财富模拟组合

用户后续会提供东方财富模拟组合操作 Skill、接口和 API key。系统接入前不得假设具体接口形态。

模拟组合在本项目中被定义为真实持仓的镜像源。用户负责手动定期校准，使模拟组合尽量接近真实账户持仓。

系统读取模拟组合时必须记录：

- 数据来源。
- 拉取时间。
- 组合更新时间。
- 用户最近一次校准时间。
- 股票代码、名称、数量、可用数量、成本价、市值等字段。

如果模拟组合数据不完整或校准时间过旧，系统应在建议中明确提示数据风险。

## 3. 东方财富自选置顶

自选置顶股票用于首版候选买入观察。系统不扫描全部自选股，只使用置顶集合，以降低噪音和数据压力。

自选置顶股票应保留排序或优先级信息，便于策略在多候选股票中排序。

## 4. AkShare

AkShare 是首版公开市场数据的首选来源。可用于：

- 日线、分钟线、历史 K 线。
- 实时或准实时行情。
- 成交额、成交量、涨跌幅。
- 资金流数据。
- 板块、指数、市场环境数据。

AkShare 数据可能存在延迟、字段变化、接口不可用或限频问题。adapter 必须显式处理异常，不得让错误数据静默进入策略。

## 5. 本地缓存

系统应缓存以下信息：

- 持仓镜像快照。
- 自选置顶快照。
- 行情快照。
- 历史行情数据。
- 收盘计划。
- 盘中触发结果。
- 最终建议。
- 推送结果和人工反馈。

缓存必须保留数据时间戳和拉取时间。复盘时应能区分“市场数据发生时间”和“系统获取时间”。

## 6. 密钥管理

API key、token、cookie、账号信息和任何敏感凭据不得提交到 git。

实现阶段应使用本地环境变量、未入库配置文件或系统密钥管理方式读取敏感信息。示例配置只能包含占位符。

## 7. Adapter 约束

所有外部数据源都应通过 adapter 访问。策略、风控和推荐模块不得直接调用 AkShare 或东方财富接口。

adapter 输出应尽量转成项目内部统一模型，避免外部字段名泄漏到核心逻辑。
