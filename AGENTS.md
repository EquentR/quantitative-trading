# Harness Agent 工作规约

## 项目身份

本项目是个人 A 股短线量化决策辅助系统。系统只输出可解释建议，不自动下单，不控制真实交易客户端，不绕过人工确认。

任何 agent 在本仓库工作时，必须优先遵守 `docs/project-spec.md`、`docs/trading-policy.md`、`docs/data-sources.md` 和 `docs/recommendation-contract.md`。

## 安全边界

禁止实现或引入以下能力，除非用户在明确理解风险后另行调整项目规约：

- 自动真实下单。
- 模拟点击真实交易客户端完成交易。
- 读取或提交真实账户密码、token、cookie、API key。
- 将密钥、账号、持仓敏感数据提交到 git。
- 输出缺少风险和失效条件的买卖建议。
- 将系统输出描述为确定性收益或保证正收益。

## 运行形态要求

项目实现必须同时支持后台常驻服务和 CLI 操作入口。

后台和 CLI 必须共享核心逻辑，包括数据 adapter、手动持仓台账、股票池、策略、风控、建议生成、通知和审计日志。不得为两个入口维护两套不一致逻辑。

## 开发纪律

- 先阅读相关 docs，再修改代码或规约。
- 对策略、风控、数据源、消息契约的修改必须同步更新 docs。
- 外部数据源必须通过 adapter 接入，核心策略不得直接依赖第三方接口字段。
- 真实持仓、成本、数量和可用数量必须以手动持仓台账为准。
- 东方财富模拟组合不得作为真实持仓来源，只能作为后续模拟操作和策略演练层。
- 策略输出必须可解释，优先规则策略，不把黑盒模型作为首版核心。
- 风控优先于策略信号。
- 任何建议必须包含数据时间戳、理由、风险、失效条件和仓位约束。
- 涉及交易逻辑、风控逻辑、数据契约的代码变更必须配套测试或可重复验证步骤。

## 当前首版范围

- 持仓权威源：手动持仓台账。
- 股票池：持仓 + 东方财富自选置顶。
- 行情来源：AkShare 为首选公开数据源。
- 决策节奏：收盘计划 + 盘中触发。
- 推送：本地控制台和结构化日志优先，后续接 Telegram、飞书、钉钉，邮件兜底。
- 风险偏好：均衡偏进取，但硬性风控优先。
- 东方财富模拟组合：后续作为自动模拟操作决策层，不作为真实持仓来源。
